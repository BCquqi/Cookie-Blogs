## 定义

深度优先搜索 (Depth First Search) 是一种很常用的搜索逻辑和算法思路，主要运行逻辑就是将一条可行的路走到底 (即不能走了或已经满足条件了)，检验是否满足条件并做相应答案更新，每一个分支挨个尝试 (分支不重复)。

根据这样的逻辑描述，这个思路的朴素变式量大约可以使用加法 / 乘法原理 (即组合) 来计算。

举个例子，从最开始的一个状态，我们可以进行操作改变状态。状态 a 可以操作为状态 b 或 c，b 可以操作为 d 或 e，c可以操作为 f 或 g，走到 d, e, g 状态都算成功，那么朴素情况下我们需要尝试的路线有: 

```
a -> b -> d (success)
a -> b -> e (success)
a -> c -> f (failed)
a -> c -> g (success)
```

显而易见，路线分支总量就是 $2 \times 2 = 2 ^ 2 = 4$，成功路线数量 $=$ 终点数量。发现这种路线可以抽象为一棵树。

正是因为其接近排列组合的路线数量，很多题目的暴力解法使用dfs进行穷举，可以树状枚举决策数量，情况数明显比线性枚举深入。

## DFS 树形抽象结构
从上面的例子可以清晰看出，DFS 的所有决策过程可以完全抽象为一棵搜索树：初始状态为树的根节点，每一次状态转移、每一次选择，就是当前节点向下延伸出的子节点，一条完整的搜索路径就是从根节点到叶子节点的一条树链。

在上述案例中，根节点为状态 a，a 分出 b、c 两个子节点，b、c 再各自分出两个子节点，所有叶子节点 d、e、f、g 就是本次搜索的所有最终状态。DFS 的核心行为，就是遍历整棵搜索树的所有树链，不遗漏、不重复地枚举每一种决策情况。

区别于广度优先搜索（层序遍历树），深度优先搜索的遍历规则是：优先纵向深挖，再横向拓展。遇到一个分支时，会一直递归深入到叶子节点（无路可走），回溯后再尝试当前节点的下一个分支，直至所有分支全部遍历完成。这也是“深度优先”名字的由来。

## 原理
DFS 绝大多数场景依托递归 + 回溯实现，这是算法的核心精髓，也是区别于普通暴力枚举的关键。

### 递归深入
程序从初始状态出发，选中一个可行分支，更新当前状态，然后递归进入下一层，继续向更深层级搜索，直到触发终止条件（到达叶子节点、满足题目答案条件、无路可走）。

### 回溯重置
当一条路径搜索完毕（无论成功或失败），程序会回到上一层节点，撤销当前分支的状态修改，还原成进入该分支前的初始状态，随后尝试下一个未遍历的分支。

回溯是 DFS 可以重复利用状态、枚举所有情况的核心。如果缺少回溯操作，状态会被上一次搜索污染，导致后续分支的枚举结果出错，出现漏解、重解的问题。

## 搜索终止条件
DFS 必须设置明确的终止条件，否则会出现无限递归、栈溢出的问题，常见终止条件分为三类：
- 边界终止：搜索到达地图边界、状态枚举到最大层数、无后续可操作分支，无路可继续深入。
- 答案终止：当前状态满足题目要求，记录答案、更新最优解，提前结束当前路径搜索。
- 剪枝终止：当前路径已经不可能得到合法解、或结果一定劣于已有最优解，直接提前终止该分支，避免无效搜索。

## 朴素 DFS 的优缺点
### 优点
- 逻辑直观：完全贴合枚举思维，基于树形决策，思路清晰，容易理解和手写代码。
- 适用性广：可解决排列、组合、路径查找、状态枚举、子集生成等各类枚举型问题。
- 空间开销小：仅需递归栈存储当前路径状态，无需额外开辟大规模队列空间，相比 BFS 空间更优。

### 缺点
- 时间复杂度高：朴素 DFS 会枚举所有分支，时间复杂度对应搜索树的总节点数，在数据规模较大时会出现超时。
- 存在递归限制：递归深度过大会触发程序栈溢出错误，无法处理深度极大的搜索场景。

## DFS 剪枝
正是因为朴素 DFS 存在大量无效枚举，实际解题中都会搭配剪枝操作优化算法效率。所谓剪枝，就是在搜索过程中提前判断：如果当前分支无论如何深入，都无法得到合法解或更优解，就直接截断该分支的后续搜索，剪掉搜索树中无用的枝干。

常见的剪枝思路包括：可行性剪枝、最优性剪枝、重复状态去重、上下界限定剪枝等。剪枝可以极大减少枚举次数，让 DFS 可以适配更大的数据范围，也是算法竞赛、刷题中 DFS 的核心考点。

## 常见适用场景
结合 DFS 树状枚举、深度遍历的特性，其经典使用场景如下：
- 排列组合问题、子集、子序列枚举
- 网格地图的路径搜索、连通块遍历（迷宫问题、岛屿问题）
- 全排列、合法方案计数类问题
- 状态回溯类问题（八皇后、数独填充）
- 树形结构的遍历、子树信息统计

## 简易通用代码模板
这里提供最通用的递归回溯 DFS 模板，适配绝大多数枚举、搜索类题目：

```cpp
void dfs(int step)
{
	if (终止条件) {
		计数;
		return;
	}
	策略更改;
	dfs(step + 1);
	回溯;
}
```
