# 广度优先搜索（BFS）完整介绍
## 定义 & 原理
广度优先搜索（Breadth First Search，BFS）是与深度优先搜索相对的经典搜索算法，是图论、搜索类题目中最核心的算法思路之一。其核心运行逻辑为**逐层向外扩展、由近及远遍历**，不深度深挖单条分支，优先遍历当前所有可达的最近状态，再迭代拓展下一层状态。

不同于 DFS 走到底再回溯的逻辑，BFS 放弃了深度枚举，采用**广度横向拓展**的搜索思路。该特性让 BFS 天然适配层级化问题，同样可以将搜索过程抽象为树形结构，但遍历方式与 DFS 完全相反。

沿用上文 DFS 的同款状态案例：状态 a 可以拓展为状态 b、c；状态 b 可以拓展为 d、e；状态 c 可以拓展为 f、g，d、e、g 为成功状态。

DFS 是单路径走到底，而 BFS 的遍历顺序严格遵循层级：
```
第一层（起点）：a
第二层（一步可达）：b、c
第三层（两步可达）：d、e、f、g
```

完整遍历顺序：
```
a -> b -> c -> d -> e -> f -> g
```

可以直观发现：BFS 不会优先走完一条路径，而是**遍历完当前所有分支，再进入下一层分支**。这种层序遍历的特性，是 BFS 最核心、最独特的算法优势。

## BFS 树形抽象结构
和 DFS 一致，BFS 的所有决策过程同样可以抽象为**搜索树**，但遍历逻辑截然不同。

搜索树中，根节点为初始状态，每一层节点对应**固定操作步数/固定距离**的所有状态：
- 第 0 层：初始起点（步数为 0）
- 第 1 层：所有一步可达的状态
- 第 2 层：所有两步可达的状态

BFS 的本质就是**树的层序遍历**。它会完整遍历完当前层级的所有节点，才会进入下一层级的搜索。正因为严格按层级递进，**首次抵达目标状态时，对应的路径一定是最短路径、最少操作步数**，这是朴素 DFS 不具备的核心特性。

## 队列迭代
DFS 依靠「递归+回溯」实现，而 BFS 全程采用**队列（Queue）迭代**实现，无需递归，不存在栈溢出风险，是迭代型搜索算法的代表。

### 核心运行流程
1. **初始化状态**：将初始起点加入队列，同时标记该状态已访问，防止重复入队；
2. **逐层出队**：取出队列头部的当前状态，判断是否满足终止条件；
3. **拓展新状态**：遍历当前状态所有合法操作，生成下一层新状态；
4. **去重入队**：未访问、合法的新状态标记后入队；
5. **循环迭代**：重复上述步骤，直至队列为空或找到目标状态。

## BFS 终止条件
BFS 逻辑简洁，终止条件清晰，主要分为两类：
- **目标终止**：遍历过程中首次匹配到目标状态，立即终止搜索，此时得到的解为最优最短解；
- **遍历终止**：队列为空，所有可达状态全部遍历完毕，无有效目标解，搜索结束。

一般来说的 BFS 不需要终止条件，不像 DFS，它会自己结束。

## 朴素 BFS 的优缺点
### 优点
- **天然求解最短路径**：无权图、网格迷宫、最少操作步数问题的最优解，首次到达即为最优解；
- **无递归限制**：采用迭代实现，没有递归深度限制，不会出现栈溢出问题；
- **层级逻辑清晰**：严格按步数、距离分层，便于统计层级信息、扩散范围；
- **稳定性强**：遍历逻辑固定，无漏解、重解问题。

### 缺点
- **空间开销较大**：需要存储整层的所有节点，当搜索树宽度极大时，队列占用内存远高于 DFS；
- **不适合枚举所有方案**：擅长找最优解，不适合全量枚举、排列组合、回溯类问题；
- **仅适配无权图**：朴素 BFS 仅能解决无权路径问题，无法处理带权路径搜索。

## BFS 常见优化方式
针对朴素 BFS 空间高、遍历效率有限的问题，算法解题中常用优化手段：
- **双向 BFS**：从起点和终点同时双向层级拓展，两端相遇即停止，大幅减少搜索节点数量，效率翻倍；
- **0-1 BFS**：适配边权仅为 0 或 1 的特殊图，使用双端队列优化，替代复杂的最短路算法；
- **状态压缩**：对复杂状态进行压缩存储，减少标记数组空间开销；
- **边界剪枝**：提前过滤越界、非法、无解的状态，避免无效入队操作。

## 常见适用场景
结合 BFS 层序遍历、最短路径的核心特性，其经典使用场景如下：
- 无权图、网格迷宫的**最短路径、最少步数**求解；
- 洪水填充、连通块统计（岛屿数量、区域遍历）；
- 二叉树、搜索树的层序遍历；
- 最少操作次数类问题（状态变换、谜题求解）；
- 拓扑排序（Kahn 算法）；
- 多源扩散、多起点同步搜索问题。
